# 文件导入审核中心 Update 规划

创建时间：2026-05-06 16:16:01 CST

## 修改目标

把现有 FAQ 管理扩展为“文件导入审核中心”：用户上传文件，系统保存原件记录，解析文档结构，按结构切块，调用 AI 生成候选问答，用户逐条编辑、筛选、确认，最后保存为正式 FAQ，并延续现有“保存正文”和“生成 embedding”分离的工作流。

## 初步范围

- 文件管理：上传、列表、状态、下载原件、删除或归档。
- 解析任务：支持待解析、解析中、解析失败、待审核、已完成等状态。
- 文档切块：Word/PDF 按标题、段落、页码；xlsx 按表头和行；Markdown 按标题段落；微信聊天记录按时间和对话轮次。
- AI 候选问答：从切块内容生成候选问题、答案、相似问法、分类、标签、来源证据。
- 人工审核：候选问答支持编辑、合并、忽略、保存；默认不直接进入可检索状态。
- 向量生成：复用现有单条/批量 embedding 逻辑，审核保存后再生成向量。

## 影响范围

- 数据库：需要新增上传文件、解析任务、文档切块、候选问答相关表。
- 后端 API：需要新增上传、下载、解析、候选问答审核、保存到 FAQ 的接口。
- AI 解析：需要新增文档抽取、切块和问答生成模块。
- 管理 UI：需要新增“导入中心”视图，和现有 FAQ 列表形成两个主要工作区。
- 测试：需要覆盖文件类型识别、切块策略、AI 输出解析、候选问答审核保存、上传限制。

## 预期效果

- 用户可以把平台手册、注意事项、SOP、FAQ 表格、微信聊天记录导入系统。
- AI 只生成“候选问答”，不会绕过人工审核直接污染正式知识库。
- 每条保存后的 FAQ 保留来源证据，后续可追溯到文件、页码、章节、表格行或聊天时间。
- 后续 RAG 检索仍只使用审核通过且 embedding ready 的内容。

## 待用户确认

1. 第一期文件类型优先级：是否先做 `xlsx + Markdown + PDF`，Word 放第二步，还是四类一起做？
2. 上传文件是否只需要本地磁盘存储，还是要预留对象存储接口？
3. 审核界面是更偏“左文件/中切块/右候选问答”的三栏工作台，还是“文件列表 -> 详情页 -> 候选问答表格”的两层页面？
4. AI 生成问答时，是否需要一次输出“标准问答 + 相似问法 + 引用证据 + 置信度”？
5. 微信聊天记录 Markdown 的格式是否有固定样例？如果有，需要放入 `user-materials/`。
