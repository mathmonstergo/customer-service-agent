# 文件导入审核中心 Update 规划

创建时间：2026-05-06 16:16:01 CST

## 修改目标

把现有 FAQ 管理扩展为“文件导入审核中心”：用户通过通用“上传文件”入口上传知识源文件，系统保存原件记录并自动识别格式。第一期只解析 Markdown 微信聊天记录，按时间窗口切块，调用 AI 生成候选问答，用户逐条编辑、筛选、确认，最后保存为正式 FAQ，并延续现有“保存正文”和“生成 embedding”分离的工作流。

## 初步范围

- 文件管理：通用上传、自动识别格式、列表、状态、下载原件。
- 解析任务：支持待解析、解析中、解析失败、待审核、已完成等状态。
- 文档切块：第一期只支持 Markdown 微信聊天记录，默认按 1 天时间范围粗切，用户可在 1-7 天之间调整，也可切换为按连续对话间隔切块；PDF、Word、xlsx 暂不解析，仅识别并显示暂不支持。
- AI 候选问答：从切块内容生成候选问题、答案、相似问法、分类、标签、来源证据、内部备注。
- 人工审核：候选问答在主工作区先展示摘要列表，点击候选或审核按钮后从右侧抽屉打开，支持编辑、忽略、保存；默认不直接进入可检索状态。
- 向量生成：复用现有单条/批量 embedding 逻辑，审核保存后再生成向量。

## 影响范围

- 数据库：需要新增上传文件、解析任务、文档切块、候选问答相关表。
- 后端 API：需要新增上传、下载、解析、候选问答审核、保存到 FAQ 的接口。
- AI 解析：需要新增文档抽取、切块和问答生成模块。
- 管理 UI：需要新增“导入中心”视图，和现有 FAQ 列表形成两个主要工作区；候选详情复用侧滑抽屉交互，不做常驻三栏详情。
- 测试：需要覆盖文件类型识别、切块策略、AI 输出解析、候选问答审核保存、上传限制。

## 预期效果

- 用户可以先把微信聊天记录 Markdown 导入系统；后续可扩展平台手册、注意事项、SOP、FAQ 表格等文件。
- 用户可以在导入审核页勾选单个或全部切块，并对选中切块批量生成候选 FAQ。
- AI 只生成“候选问答”，不会绕过人工审核直接污染正式知识库。
- 每条保存后的 FAQ 保留来源证据，后续可追溯到文件、页码、章节、表格行或聊天时间。
- 后续 RAG 检索仍只使用审核通过且 embedding ready 的内容。
