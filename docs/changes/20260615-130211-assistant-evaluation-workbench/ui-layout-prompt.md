# UI Layout Prompt

请为一个本地内部客服知识库后台设计“问答效果验收工作台”页面的高保真布局图。

背景：这是 Python + PostgreSQL + pgvector + React 的内部管理工具，不是营销页。页面用于维护标准问题集、运行单条检索评测、查看命中来源和指标，帮助团队判断知识库资料、切块、关键词、别名和检索策略是否有效。现有后台是深色工具型界面：左侧固定导航、顶部细工具栏、紧凑列表、右侧抽屉/详情，整体接近 Linear 风格，暗色背景、细边框、低饱和文字、一个蓝紫主色。

设计目标：

* 第一屏就是可操作工作台，不要 hero、欢迎页、营销文案或装饰图。
* 信息密集但清晰，适合内部人员反复验收和调参。
* 主要路由为 `/evaluation`，左侧导航新增“效果验收”并高亮。
* 页面文案使用中文，术语包含：评测用例、运行策略、期望命中、最近运行、Recall@K、MRR、Top1、候选来源、别名词典。
* 当前 MVP 只支持运行单条评测用例，不设计批量运行入口。
* 为未来 agentic RAG 预留口径：运行策略显示为 `retrieval_hybrid_v1`，但布局中可以有“运行轨迹”区域，未来可放 tool calls / reasoning trace / iteration count；当前只展示 query analysis、query rewrite、vector_count、keyword_count。

推荐桌面布局（1440x900）：

1. 左侧应用导航沿用现有后台：
   * 宽 224px，可折叠。
   * 导航项：文档管理、FAQ 管理、智能问答、效果验收。
   * 使用 lucide 风格线性图标，效果验收可用 Activity / Gauge / ClipboardCheck 一类图标。

2. 页面顶部工具栏：
   * 左侧搜索框：placeholder “搜问题 / 标签 / 意图…”
   * 紧挨着状态 segmented control：全部、启用、禁用。
   * 中部显示轻量统计：用例数、最近运行命中率、待补期望命中。
   * 右侧主按钮“新建用例”，次按钮“别名词典”或直接打开右侧别名区域。

3. 主工作区采用三栏：
   * 左栏 340-380px：评测用例列表。每一行显示问题、标签、状态、小号最近运行指标（Recall、MRR、Top1）、最后运行时间。选中行有低调主色背景，不要厚重卡片。
   * 中栏自适应：运行结果详情。顶部显示选中问题和“运行单条”按钮；下方是紧凑指标条 Recall@K / MRR / Top1 / vector_count / keyword_count；再下方是候选来源表格或列表，列包括 rank、命中标记、source_type、source_id、chunk_id、score、channels、内容预览。期望命中的行用绿色细标记，未命中用黄色/红色弱提示。
   * 右栏 320-360px：检查器。上半部分是用例编辑表单：问题、意图、期望 source ids、期望 chunk ids、标签、备注、状态；下半部分是别名词典最小维护区：标准词、别名、状态、保存按钮。

4. 运行轨迹/分析区域：
   * 放在中栏指标条和候选来源之间，使用很薄的分隔和 chips。
   * 展示 intent、confidence、query_rewrite、query_terms、运行策略。
   * 不展示完整 agent 逻辑；只预留“运行轨迹”标题和单步 `hybrid retrieval`。

5. 状态与空态：
   * 无用例：左栏显示紧凑空态和“新建用例”按钮。
   * 未运行：中栏显示“选择用例后运行评测”，不要大插画。
   * 运行中：运行按钮显示 spinner，候选区保持旧结果或显示局部 skeleton，不要整页闪烁。
   * 失败：在结果区顶部显示红色细边提示，可重试。

视觉要求：

* 深色工具型 UI，背景接近 #08090a，surface #111213 / #17181a，细边框，文字层级清楚。
* 不要使用大圆角卡片堆叠；重复项可以用 6-8px 圆角列表行或轻量面板。
* 不要使用装饰性渐变、光球、bokeh、营销 banner。
* 字号克制：页面主要文字 12-14px，指标数字 16-20px，不要 hero 级大标题。
* 图标只用于提升扫描：搜索、运行、保存、标签、命中、失败、别名。
* 内容不要互相遮挡；长问题和 source id 需要截断并有稳定列宽。
* 画面请体现真实数据，不要 lorem ipsum。示例问题可用：
  * “报告导出失败怎么办？”
  * “客户说测评报告没有生成怎么处理？”
  * “在哪里配置团体报告下载权限？”
  * 别名：“报告 = 测评报告、团体报告”，“账号 = 账户、登录名”。

请输出一张桌面端高保真 UI 布局图，重点展示信息层级、三栏比例、运行结果详情和状态呈现。
