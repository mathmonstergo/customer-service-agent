# FAQ 管理页子视图与切片查看调整计划

创建时间：2026-05-13 15:53:13 CST

## 修改目标

在 FAQ 管理页顶部增加三个同级按钮：`FAQ 列表`、`FAQ 自动生成`、`FAQ 审核`。用户切换后仍处在 FAQ 管理工作区内，但内容分别对应标准问答列表、从已解析文件生成候选 FAQ、候选 FAQ 人工审核。选中样式使用简约下划线，并做成中间较粗、两端较细的视觉效果。

同时调整 FAQ 自动生成和文档切片查看体验：

- FAQ 自动生成页不再提供上传/导入文件按钮。
- 文件来源改为“选择文件”，只能选择已经解析并有切片的文件。
- 切片详情/切片查看区域只保留切片原文，不再在正文框内展示序号、段内容、正文、可用于 FAQ 生成等辅助信息。
- 查看切片内容时，滚动应发生在切片查看框内部，不依赖整个详情弹窗滚动。

## 影响范围

- 静态页面：`customer_service_agent/static/admin.html`
- 前端交互：`customer_service_agent/static/admin.js`
- 前端样式：`customer_service_agent/static/admin.css`
- 静态结构测试：`tests/test_admin_table_layout.py`

本次优先不改数据库 schema，不新增上传接口，不改变候选 FAQ 保存和 embedding 生成逻辑。

## 具体步骤

1. 在 FAQ 管理页主内容头部把 `FAQ 列表` 做成子视图按钮，并新增 `FAQ 自动生成`、`FAQ 审核` 两个按钮。
2. 增加前端状态字段记录当前 FAQ 子视图，切换按钮时分别显示标准 FAQ 表格、自动生成工作台、候选审核工作台。
3. 将现有 `importWorkspace` 的解析块视图接入 `FAQ 自动生成`，将候选 FAQ 视图接入 `FAQ 审核`，并保持现有生成任务、候选审核、来源块筛选能力。
4. 移除 FAQ 自动生成页的上传文件 input/按钮，改为文件选择控件和已解析文件列表；前端仅展示 `needs_review`、`completed` 或已有切片的文件。
5. 修改空状态和搜索文案，明确用户需要先在文档管理页上传并解析文件，再回到 FAQ 自动生成选择文件。
6. 简化文档详情抽屉里的切片内容渲染：`renderDocumentChunkContent` 只输出原文容器，不再输出标题、元信息和标签。
7. 为文档切片查看区设置固定内部高度和 `overflow-y: auto`，让长切片内容在查看框内滚动。
8. 先补静态结构测试，确认按钮、选择文件控件、移除上传入口、只渲染原文、局部滚动样式都被覆盖；再实现前端改动。

## 预期效果

- 用户在 FAQ 管理页内就能完成“列表维护 / 自动生成 / 审核保存”的完整流程，不需要理解单独的导入审核入口。
- FAQ 自动生成只能基于已解析文件，避免从该页面重新上传原件造成“文档管理”和“FAQ 生成”职责混淆。
- 切片详情阅读更干净，只看原始内容；序号、段数、正文标签等辅助信息保留在列表或标题区域，不进入原文框。
- 长切片不再把整个详情弹窗撑出滚动，阅读时只滚动切片查看框。

## UI Prompt

用于和另一个 AI 沟通 UI 布局的 prompt：

> 设计一个本地客服知识库管理后台的 FAQ 管理页面，整体是工具型、信息密集但清晰的管理界面。页面顶部保持现有系统导航，进入 FAQ 管理后，在主内容上方放三个横向按钮：FAQ 列表、FAQ 自动生成、FAQ 审核。按钮简洁无卡片，当前选中项使用中间较粗、两端较细的蓝色下划线，不使用营销式装饰。FAQ 列表显示标准 FAQ 表格和新建/刷新按钮；FAQ 自动生成显示左侧“选择文件”区域，只列出已解析且有切片的文件，不提供上传文件按钮，右侧显示解析块列表、生成进度和切片原文预览；FAQ 审核显示候选 FAQ 表格、筛选器和右侧审核抽屉。切片原文预览框只显示原始文本，框体自己有上下滚动条，不让整个弹窗或页面承担长文本滚动。
