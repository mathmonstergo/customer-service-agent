# 用户确认记录

确认时间：2026-05-12 10:57 左右。

确认内容：

- 暂停本地 Docker 版 MinerU 安装。
- 使用 MinerU 第三方服务作为当前文档解析方案。
- 当前阶段先不做本地部署接口选择，MinerU 云端接口固定在代码层，设置页只填写 Token。
- 项目层保留可选再处理开关，参考 `frondesce/mineru-kb-packager` 处理 MinerU 输出。
- 后续增加独立设置页，用于管理解析器、LLM 提供商、Embedding 等配置。
- 用户提供了 MinerU Token；该 Token 按敏感配置处理，不写入代码或文档。

补充确认：

- 导入审核里点击解析块时，先查看切片真实内容，不应直接跳到该切片的候选 FAQ。
- 文档切片不默认等价于 FAQ；用户需要能按需对当前切片或所选切片生成候选 FAQ。
- 设置中心内 API Key / Token 需要能通过显示按钮查看真实值，图标采用常见的眼睛、眼睛关闭和复制图标。
- Chat Model 不能只用下拉框，需要支持直接输入任意 OpenAI-compatible 模型名，同时保留常用模型候选。
- 设置弹窗内出现的保存、测试或错误提示不能被玻璃遮罩挡住。
- API Key / Token 隐藏态不应按真实长度显示圆点，统一固定显示 16 个黑点。
- 设置页不需要“保存并测试”按钮；保存时只校验必要内容并写回本地配置，不对外部 API 做连通性测试。
- 设置页切换选项后如果改回原值，不应继续提示存在未保存修改。
- 后续要做多租户，设置页不应直接写 `.env`；租户级解析器、LLM、Embedding 和密钥配置应隔离保存。
- MinerU 地址不让用户填写；使用官方批量文件接口，URL 固定在代码层维护。
- MinerU KIE SDK 属于另一个功能方向，本次不接入。
- 知识库顶层需要轻量化，不做复杂数据大屏；主页只做搜索、入口和少量待处理状态。
- 当前“标准问答 / 导入审核”关系调整为“知识库主页 -> FAQ 管理 / 文档管理”；文档管理承载上传、MinerU 解析、切片查看和后续从切片生成候选 FAQ。
- 知识库主页 UI 需要复刻用户提供的截图效果：入口卡片第一次点击展开顶层悬浮面板，不影响其他元素排列；展开状态下再次点击卡片本体进入对应页面。
- 文档管理入口不能再指向原“导入审核 / 生成 FAQ”页面；该旧页面实际属于 FAQ 管理下的“从切片生成候选 FAQ”子功能。
- 正式文档管理页需要重新设计：文档导入默认只存储原件，不立即解析；点击某个文件行后用顶层侧拉抽屉展示详情操作、解析/重解析和切片查看，不影响初始页面布局。
- 文档管理 UI 可参考用户提供的表格 + 右侧侧拉详情截图实现。
- 没有失败切片时不展示失败切片区域；MinerU 当前更适合按文件级状态和错误信息处理失败。
- MinerU 查询状态是动态的，前端需要兼容 `state` 和 `extract_progress`，在解析中展示进度并在完成后刷新切片。
