# 导入审核工作台重排计划

创建时间：2026-05-08 11:12:45 CST

## 修改目标

把现有“导入审核”前端从切块表格和候选列表上下堆叠，调整为更贴近设计图的工作台：左侧仍管理导入文件，主工作区通过“解析块 / 候选 FAQ”二级视图分别承载解析生成和候选审核。保持小团队轻量工具定位，不引入复杂权限、审批流或 BI 大屏。

## 影响范围

- 静态页面：`customer_service_agent/static/admin.html`、`admin.css`、`admin.js`。
- 后端 API：新增按导入文件列出全部候选 FAQ 的轻量接口，支持候选 FAQ 视图不依赖单个切块；候选保存到标准问答后立即生成 embedding。
- 数据库读写：新增文件级候选查询方法。
- 测试：补充静态结构测试、API 单元测试和现有回归测试。

## 具体步骤

1. 在导入审核主工作区增加二级 tab：`解析块` 与 `候选 FAQ`。
2. 解析块视图按第一张效果图优先一比一复刻：顶部文件概览、解析与生成进度、当前处理块、解析模式、重新解析、勾选切块、自动识别 FAQ、仅查看待审核和切块表格。
3. 候选 FAQ 视图改为文件级候选列表，展示状态、标准问题、分类、重复度、置信度、来源块、更新时间和操作。
4. 右侧候选审核抽屉复用当前字段，并优化文案为“保存到标准问答 / 忽略 / AI 保守改写 / 查看来源”。
5. 生成任务完成后刷新当前文件的切块和候选汇总。
6. 候选 FAQ 保存到标准问答时自动生成 embedding；若 embedding 失败，正式 FAQ 保留且记录失败状态供重试。
7. 候选审核抽屉中的“重复程度/重复度”表示候选与已有知识的重复风险，不使用“重要程度”概念。
8. 标准问答的新建/编辑抽屉同步候选审核抽屉的信息层级和视觉效果，避免同一类内容出现两套编辑体验。
9. 解析块和候选 FAQ 复用同一顶部概览框、进度框和工作区外框；切换时只替换内部指标、进度和表格内容。
10. 从解析块行进入候选 FAQ 时，带入来源块筛选；用户可在候选页清除来源筛选后查看当前文件全部候选。

## 预期效果

- 用户能更清楚地区分“解析与生成”和“候选审核”两个阶段。
- 候选审核不再必须先点某一个切块才能看到候选，适合批量处理微信聊天记录和文档资料。
- 从具体解析块进入候选 FAQ 时，默认只看该块生成的候选，减少人工审核时的上下文跳失。
- 导入链路保持轻量：文件、切块、候选、审核保存四个核心对象，不引入企业级重流程。
- 审核人员点击保存后不再额外手动生成 embedding，减少小团队维护步骤。

## 不做事项

- 不做解析页“状态范围”筛选；第一张图里的范围控件只保留为“块范围”视觉和切块定位信息。
- 不做复杂权限、多租户、审批流。
- 不让未审核候选直接进入检索；只有人工点击“保存到标准问答”后才自动生成 embedding。
- 不把“重复度”做成业务重要性评分。
