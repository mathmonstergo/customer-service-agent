# 管理后台浏览器验证与交互优化计划

## 修改目标

- 配置或使用可运行的浏览器自动化能力，真实访问本地管理后台。
- 逐项检查标准问答、AI 辅助编辑、导入审核中心里的按钮、状态提示、抽屉和批量操作是否符合预期。
- 评估页面布局是否服务于“客服知识库管理”和“文件导入审核”的核心任务。
- 对明确影响操作效率、理解成本或误操作风险的问题做小步优化。

## 影响范围

- 本地浏览器验证链路：优先使用 Playwright CLI / MCP 类能力。
- 管理后台静态页面：`customer_service_agent/static/admin.html`、`admin.css`、`admin.js`。
- 如发现后端接口状态或返回值影响前端体验，才会触及 `customer_service_agent/admin_server.py` 及相关测试。

## 具体步骤

1. 检查 Node/npm、Playwright 或 Chrome DevTools MCP 是否可用。
2. 启动本地 admin 服务，并用浏览器自动化访问 `/admin.html`。
3. 记录主要页面区域、按钮、表单、抽屉、批量操作的实际表现。
4. 按“能否减少误点、能否清楚表达状态、是否匹配知识库审核流程”判断是否需要调整。
5. 做必要的前端或接口修复，并补充对应验证。
6. 运行 Python 测试、ruff、前端语法检查和浏览器冒烟测试。

## 预期效果

- 管理后台按钮行为更可验证，关键流程有清晰状态反馈。
- 导入审核中心的操作顺序更接近真实人工审核：先选文件/分块，再生成候选，再侧边审核。
- 页面布局保持工具型、信息密集但清晰，不增加无意义装饰。

## 浏览器验证发现

- 导入审核切块表格被候选 FAQ 区域覆盖，导致单个切块勾选和全选都会被遮挡元素拦截。
- 导入审核文件列表已有文件时，切换到页面后没有自动选中文件，用户需要额外点击才能看到切块。
- FAQ 编辑抽屉存在“最小化”按钮但无实际行为。
- 候选 FAQ 抽屉底部“应用 AI 改写”“查看来源”按钮可见但未绑定行为。
- FAQ 详情在未修改时点击保存仍会触发保存流程，容易制造更新时间噪音。

## 已实施调整

- 给导入切块表格和候选列表划分固定网格区域，避免互相覆盖。
- 增加导入切块分页，表头全选只作用于当前页可见切块。
- 切换到导入审核后自动选择当前筛选下的第一个文件。
- 移除 FAQ 抽屉无效果的最小化按钮。
- 给通知按钮、筛选折叠、候选 AI 改写、查看来源补充明确行为。
- FAQ 未修改保存时只提示“没有需要保存的改动”，不再发保存请求。
- 修复标准问答页向下滚动会露出导入审核页的问题，确保隐藏工作区覆盖自身 grid 布局。
- 将候选 FAQ 生成进度从事件标签列表改为任务级进度条，只展示总进度、当前/最近处理块和聚合状态统计。
- 在导入切块表格中显示解析阶段生成的块编号，生成进度也使用同一编号定位当前块。
- 为导入和生成相关状态补充颜色区分，避免 pending、processing、generated 等状态难以扫描。
- 给生成进度条增加运行态扫光和脉冲动效；进度宽度仍只按已生成、已跳过、失败的真实完成块数计算。
- 将导入切块编号统一改为纯 `#数字`，页面文案不再显示“块 #数字”。
- 固定导入文件侧栏头部高度和下间距，避免“上传文件”和“搜索文件名”控件在视觉上贴合或重叠。
- 失败文件数为 0 时，“失败文件汇总 - 查看详情”只提示当前没有失败文件，不再切换筛选导致文件列表变空。

## 生成进度交互说明

- 当前批量生成仍为顺序处理：一个切块生成完成后才处理下一个切块。
- 如果一次选择 30 块，页面不会展示 30 个事件标签，只展示进度条和已生成、处理中、已跳过、失败四个统计。
- `done` 是任务级事件，只用于标记任务完成，不再作为单独“完成”标签追加到列表。
- 右侧当前处理位置显示解析阶段编号，例如 `当前：#2`；任务完成后显示“任务完成”。

## 需要用户确认的问题

用户已在 2026-05-07 明确确认：安装后测试每一个按钮效果和摆放布局，并允许按更符合人类操作习惯的方式调整优化。
